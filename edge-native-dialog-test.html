🚀 Microsoft Edge ネイティブ許可ダイアログテスト

LocalNetworkAccessRestrictionsEnabled Policy

📋 テスト概要

このページは、Microsoft EdgeのLocalNetworkAccessRestrictionsEnabledポリシーによるブラウザネイティブの許可ダイアログをテストします。

ポリシーが有効な場合、ローカルネットワークアクセス時にMicrosoft Edge自体が許可ダイアログを表示します。

🔍 Microsoft Edge ネイティブダイアログについて

LocalNetworkAccessRestrictionsEnabledポリシー有効時の動作:

1
ローカルネットワークアクセス要求の検出
2
Microsoft Edgeがネイティブ許可ダイアログを表示
3
ユーザーが「許可」または「ブロック」を選択
4
選択に基づいてアクセス許可/拒否
予想されるEdgeダイアログ例:
"🔒 このサイトがプライベートネットワークリソース (192.168.1.1) にアクセスしようとしています。
[ 許可 ] [ ブロック ]"

• サイト設定での許可状態
• ローカルストレージの変化
• サービスワーカーの動作

🛠️ 期待される動作

ポリシー有効時 (LocalNetworkAccessRestrictionsEnabled = true): 1. 初回アクセス: - ローカルネットワーク要求を検出 - Microsoft Edgeがネイティブ許可ダイアログを表示 - ユーザーの明示的な選択が必要 2. 許可後: - 同じサイトからの同種要求は許可 - サイト設定に許可情報が保存 - 今後のアクセスは自動許可 3. ブロック後: - 同じサイトからの要求は継続ブロック - サイト設定でブロック状態が保持 - 手動でサイト設定を変更するまで継続 ポリシー無効時 (LocalNetworkAccessRestrictionsEnabled = false): 1. 従来の動作: - 警告のみ表示（DevToolsで確認可能） - 要求は基本的に継続 - ユーザー許可は不要
