📋 テスト概要

このページは、Microsoft EdgeのLocalNetworkAccessRestrictionsEnabledポリシーの「ユーザーが明示的に許可しない限り」機能をテストします。

ポリシーが有効な場合、ローカルネットワークアクセス前に明示的なユーザー許可が必要になります。

🛠️ 実装詳細

ポリシー有効時の期待動作：
1. 自動ブロック: 許可なしの要求は即座にブロック
2. 許可プロンプト: ユーザージェスチャ後に許可ダイアログ表示
3. 明示的選択: ユーザーが「許可」または「拒否」を明示的に選択
4. 状態管理: 許可状態がセッション（またはサイト設定）で保持
5. 細かい制御: URL/ポート別の詳細な許可管理

技術実装ポイント：
• navigator.permissions.query() for permission status
• fetch() with proper error handling
• User gesture requirements for permission requests
• Custom permission UI for fallback scenarios
• Persistent permission storage mechanisms
